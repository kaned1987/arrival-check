# ✅ ArrivalCheck

ระบบเช็คชื่อแบบ self-service สำหรับงานที่ต้องเช็คชื่อหลายรอบ เช่น **ขึ้นรถบัสศึกษาดูงาน, เข้าห้องประชุม, ลงทะเบียนงานสัมมนา, เช็คชื่อเข้าคลาส**

**Stack**: HTML/JS แบบ static + Google Apps Script + Google Sheets + GitHub Pages

---

## 🎯 จุดเด่น

- ผู้ใช้ **เช็คชื่อเอง** ด้วย 4 หลักท้ายเบอร์โทรเป็น PIN กันคนกดแทน
- รองรับ **เช็คชื่อหลายรอบ** ในงานเดียวกัน (ครั้งที่ 1, 2, 3 ... สูงสุด 40)
- **Dashboard real-time** สำหรับผู้ดูแล พร้อมเบอร์โทรกดโทรออกได้
- กันกดซ้ำในครั้งเดียวกัน
- Export CSV เก็บหลักฐาน

---

## 🚀 ขั้นตอนติดตั้ง (ครั้งแรกครั้งเดียว)

### ① สร้าง Google Sheet + วาง Apps Script

1. ไปที่ [sheets.new](https://sheets.new) สร้าง Sheet ใหม่ ตั้งชื่อ **"ArrivalCheck"**
2. เมนู **Extensions → Apps Script** → ลบโค้ดเดิมในไฟล์ `Code.gs`
3. คัดลอกเนื้อหาทั้งหมดจากไฟล์ `Code.gs` ในโฟลเดอร์นี้ ไปวาง
4. กด **บันทึก** (Ctrl+S)
5. เลือกฟังก์ชัน `setupSheets` ในดรอปดาวน์ด้านบน → กด **▶ Run**
   - ครั้งแรกจะขอสิทธิ์ → Allow
   - จะสร้างชีต `Roster` + `CheckIns` พร้อม mock 10 คน

### ② Deploy Web App

1. ใน Apps Script: มุมขวาบน **Deploy → New deployment**
2. ⚙ คลิกเฟือง → เลือก **Web app**
3. ตั้งค่า:
   - **Description**: arrival-check v1
   - **Execute as**: Me (ตัวเอง)
   - **Who has access**: **Anyone** (สำคัญ — ถ้าเลือก "Anyone with Google account" จะใช้ไม่ได้)
4. กด **Deploy** → คัดลอก **Web app URL**

### ③ ใส่ URL ลงใน config.js

เปิดไฟล์ `config.js` แก้บรรทัด:

```js
const GAS_URL = 'PASTE_YOUR_WEB_APP_URL_HERE';
```

เป็น URL ที่คัดลอกมาจากขั้นตอน ②

### ④ Push ขึ้น GitHub + เปิด Pages

1. สร้าง repo ใหม่บน GitHub เช่น `arrival-check`
2. อัปโหลด 4 ไฟล์: `index.html`, `dashboard.html`, `config.js`, `README.md`
3. ไปที่ **Settings → Pages**
   - **Source**: Deploy from a branch
   - **Branch**: `main` / **folder**: `/ (root)` → Save
4. รอ 1-2 นาที จะได้ลิงก์:
   - หน้าผู้ใช้: `https://<your-username>.github.io/arrival-check/`
   - หน้าผู้ดูแล: `https://<your-username>.github.io/arrival-check/dashboard.html`

## 📱 การใช้งานจริง

### สำหรับผู้ใช้ (นิสิต/ผู้เข้าร่วม)
1. เปิดลิงก์หน้าผู้ใช้ (แนะนำทำ QR code ติดในจุดที่จะเช็คชื่อ)
2. เลือก **ครั้งที่** ตามที่ผู้ดูแลบอก
3. ค้นหาและกดเลือกชื่อตนเอง
4. กรอก **4 หลักท้ายเบอร์โทร** ของตนเอง
5. กด **ยืนยันการเช็คชื่อ** → เห็นหน้าจอ ✓ พร้อมปุ่มกลับหน้าแรก

### สำหรับผู้ดูแล
1. เปิดลิงก์ dashboard บนมือถือ/แท็บเล็ต
2. เลือกครั้งที่ต้องการดู → จะ auto-refresh ทุก 15 วินาที
3. คนที่ยังไม่เช็คชื่อจะอยู่บนสุด **กดที่เบอร์โทรเพื่อโทรออกได้ทันที**
4. กด **Export CSV** เพื่อเก็บหลักฐาน

## ✏️ แก้ไขรายชื่อจริง

1. เปิด Google Sheet → ชีต **Roster**
2. แก้ไข/เพิ่ม/ลบรายชื่อในตารางได้เลย (กรอกครบ 4 คอลัมน์: id, ชื่อ, สาขา, เบอร์โทร)
3. **เบอร์โทรต้องเป็น 10 หลักไม่มีขีดคั่น** เช่น `0812345678`
4. ไม่ต้อง deploy ใหม่ — refresh หน้าเว็บได้เลย

## 🔄 ใช้กับงานอื่น

ระบบนี้ออกแบบให้ generic — แค่เปลี่ยน Roster ก็ใช้ได้กับงานอื่น เช่น:
- 🚌 เช็คชื่อขึ้นรถบัสศึกษาดูงาน (ครั้งที่ 1 = ขาไป, 2 = แวะพัก, 3 = ขากลับ)
- 🏛️ เช็คชื่อเข้าห้องประชุม/อบรม (ครั้งที่ 1 = เช้า, 2 = บ่าย)
- 📚 เช็คชื่อเข้าคลาส (ครั้งที่ = สัปดาห์)
- 🎓 ลงทะเบียนงานสัมมนา/พิธีมอบรางวัล

แต่ละงานสร้าง Sheet แยกกัน + Deploy GAS แยกกัน + repo แยกกัน หรือใช้ Sheet เดียวแล้วล้าง CheckIns ก่อนเริ่มงานใหม่ก็ได้

## ⚠️ หมายเหตุ

- **Dashboard ไม่มีรหัสผ่าน** ใครเจอลิงก์ก็เปิดได้ เก็บลิงก์ไว้ใช้ผู้ดูแลเท่านั้น (มีเบอร์โทร ไม่ควรเผยแพร่)
- **GAS quota**: ฟรี 20,000 calls/วัน — เกินพอสำหรับ 1 งาน
- **เปลี่ยน MAX_ROUND**: แก้ใน `config.js` ได้ถ้าอยากมากกว่า 40 ครั้ง
